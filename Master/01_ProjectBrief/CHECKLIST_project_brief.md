# ЧЕКЛИСТ: БРИФ ПРОЕКТА

Используйте этот чеклист для проверки полноты заполнения брифа проекта.

## 1. КОНТЕКСТ БИЗНЕСА

- [ ] Название компании/проекта указано
- [ ] Отрасль и тип рынка (B2B/B2C) определены
- [ ] Географический охват указан
- [ ] УТП четко сформулировано (в 1-2 предложениях)
- [ ] Перечислено минимум 3 конкурентных преимущества
- [ ] История компании записана (для established бизнесов)
- [ ] Стадия бизнеса определена

## 2. ЦЕЛИ ПРОЕКТА

- [ ] Основная бизнес-цель четко определена
- [ ] Перечислены 2-3 конкретные проблемы для решения
- [ ] Указана целевая конверсия в %
- [ ] Указан ожидаемый трафик (посетителей/месяц)
- [ ] Определены показатели вовлеченности (время, глубина, bounce rate)
- [ ] **KPI заданы в цифрах:**
  - [ ] Количество заявок/лидов в месяц
  - [ ] Количество звонков (если применимо)
  - [ ] Количество продаж
  - [ ] Средний чек
  - [ ] Ожидаемый ROI
- [ ] Сроки достижения первых результатов указаны
- [ ] Сроки выхода на плановые показатели указаны

## 3. ЦЕЛЕВАЯ АУДИТОРИЯ

### Демография
- [ ] Возрастной диапазон указан
- [ ] Пол определен
- [ ] Географическое расположение определено
- [ ] Уровень дохода указан (диапазон)
- [ ] Уровень образования указан
- [ ] Для B2B: должности и роли указаны
- [ ] Есть описание вторичной ЦА (если применимо)

### Психография
- [ ] Перечислены интересы и хобби (минимум 3)
- [ ] Определены ценности и убеждения (минимум 3)
- [ ] Описан образ жизни (детальный параграф)
- [ ] Описан процесс принятия решений
- [ ] Указано, кто влияет на решение о покупке
- [ ] Указано время на принятие решения

### Боли и потребности
- [ ] Перечислены минимум 3 основные боли (с подробным описанием)
- [ ] Указаны минимум 3 потребности
- [ ] Разделены needs vs wants

### Возражения
- [ ] Перечислены минимум 3 основных возражения
- [ ] Для каждого возражения есть план, как его закрыть
- [ ] Перечислены минимум 3 страха аудитории

### Онлайн-поведение
- [ ] Указаны социальные сети, где присутствует ЦА
- [ ] Описано, как ЦА ищет информацию
- [ ] Даны примеры поисковых запросов
- [ ] Указано, где ЦА ищет отзывы
- [ ] Описано, что влияет на решение о покупке

### Устройства и каналы
- [ ] Распределение Desktop/Mobile/Tablet указано в %
- [ ] Сумма процентов = 100%
- [ ] Перечислены источники трафика с % распределением
- [ ] Сумма источников трафика = 100%

## 4. КОНКУРЕНТНЫЙ АНАЛИЗ

- [ ] Указано минимум 3 прямых конкурента
- [ ] Для каждого конкурента указан URL сайта
- [ ] Для каждого конкурента:
  - [ ] Перечислены сильные стороны (минимум 3)
  - [ ] Перечислены слабые стороны (минимум 3)
  - [ ] Указано, что можно перенять
- [ ] Определены минимум 3 пробела на рынке (возможности)
- [ ] Каждая возможность имеет описание, как её использовать

## 5. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### Платформа
- [ ] Выбрана конкретная платформа
- [ ] Указана причина выбора платформы
- [ ] Определена необходимость CMS
- [ ] Определен уровень доступа для клиента

### Интеграции
- [ ] Перечислены все обязательные интеграции
- [ ] Для каждой интеграции указано:
  - [ ] Название системы
  - [ ] Что интегрируем
  - [ ] Кто ответственный за настройку
- [ ] Определены желательные интеграции
- [ ] Указаны системы аналитики (GA4, Метрика минимум)

### Функции
- [ ] Перечислены все MUST HAVE функции (минимум 5)
- [ ] Перечислены SHOULD HAVE функции
- [ ] Перечислены COULD HAVE функции
- [ ] Каждая функция четко описана

### Ограничения
- [ ] Указаны технические ограничения
- [ ] Указано, что запрещено использовать
- [ ] Определены требования безопасности

## 6. БЮДЖЕТ И СРОКИ

### Бюджет
- [ ] Указан минимальный бюджет
- [ ] Указан оптимальный бюджет
- [ ] Указан максимальный бюджет
- [ ] Определена гибкость бюджета
- [ ] Бюджет распределен по статьям расходов
- [ ] Распределение бюджета в % = 100%
- [ ] Заложен резерв на непредвиденные расходы (минимум 10%)

### Сроки
- [ ] Все этапы проекта расписаны в таблице
- [ ] Для каждого этапа указана длительность
- [ ] Для каждого этапа указаны даты начала и окончания
- [ ] Для каждого этапа назначен ответственный
- [ ] **Дата запуска четко определена**
- [ ] Указана причина критичности даты запуска (если применимо)
- [ ] Перечислены промежуточные критические даты
- [ ] Определены условия оплаты с процентами и суммами

## 7. КОМАНДА ПРОЕКТА

### Со стороны агентства
- [ ] Назначен Project Manager (с контактами)
- [ ] Назначен UI/UX Designer
- [ ] Назначен Developer
- [ ] Назначен Copywriter (если нужен)
- [ ] Назначен QA Specialist

### Со стороны клиента
- [ ] Указан основной контакт (с должностью, email, телефоном)
- [ ] Указано лицо, принимающее решения
- [ ] Указан ответственный за контент
- [ ] Указан технический специалист (если нужен)

### Процесс
- [ ] Определен процесс согласования дизайна
- [ ] Определен процесс согласования контента
- [ ] Определено, кто принимает финальное утверждение
- [ ] Установлено максимальное время на согласование

## 8. ДОПОЛНИТЕЛЬНАЯ ИНФОРМАЦИЯ

- [ ] Приложены минимум 3 сайта-референса
- [ ] Для каждого референса указано, что именно понравилось
- [ ] Записаны особые пожелания клиента
- [ ] Определены минимум 2 основных риска проекта
- [ ] Для каждого риска:
  - [ ] Оценена вероятность
  - [ ] Оценено влияние
  - [ ] Есть план митигации
- [ ] Определены критерии успеха проекта (минимум 5)

## 9. УТВЕРЖДЕНИЕ

- [ ] Бриф полностью заполнен
- [ ] Бриф проверен на наличие противоречий
- [ ] Бриф согласован со всеми stakeholders
- [ ] Подпись клиента получена

- [ ] Даты утверждения проставлены
- [ ] Бриф сохранен и расшарен всей команде проекта
