## 5. ПОЛЬЗОВАТЕЛЬСКИЕ ПУТИ (USER FLOWS)

### 5.1 Путь 1: Холодный посетитель → Лид

**Входная точка:** [Google Ads / Facebook Ads / SEO]

**Сценарий:**
```
1. Лендинг на Главную страницу (/)
   ↓
   Цель: Заинтересовать УТП
   Действие пользователя: Скроллинг, чтение
   Метрика: Время на странице > 30 сек

2. Скролл до секции Benefits
   ↓
   Цель: Показать ценность
   Действие: Понимание выгод
   Возможный выход: Если не убедительно → bounce

3. Клик на [CTA: "Узнать подробнее" / "Наши услуги"]
   ↓
   Переход на: /services/service-name

4. Страница услуги
   ↓
   Цель: Детальное описание решения
   Действие: Чтение, просмотр кейсов
   Метрика: Scroll depth > 50%

5. Клик на кейс
   ↓
   Переход на: /cases/case-name

6. Страница кейса
   ↓
   Цель: Социальное доказательство
   Действие: Видит результаты в цифрах

7. Клик [CTA: "Получить консультацию"]
   ↓
   Действие: Заполнение формы

8. Thank You Page (/thank-you)
   ↓
   Цель: Подтверждение, следующие шаги

9. Email follow-up
   ↓
   Дальнейшая коммуникация
```

**Критические точки решения:**
1. **Hero section (Главная)** - Остаться или уйти?
2. **Benefits section** - Это для меня?
3. **Страница услуги** - Это решает мою проблему?
4. **Кейсы** - Они добились результатов?
5. **Форма** - Оставить заявку или еще подумать?

**Точки возможного выхода и решения:**
- **Выход 1:** Неубедительный Hero → Улучшить заголовок, добавить социальное доказательство
- **Выход 2:** Непонятные преимущества → Добавить конкретные цифры и примеры
- **Выход 3:** Нет доверия → Добавить отзывы, логотипы клиентов, сертификаты
- **Выход 4:** Сложная форма → Упростить до 3-4 полей

### Принципы перелинковки:

1. **С главной страницы:**
   - На все основные разделы (через меню и секции)
   - На топ-3 услуги (в отдельной секции)
   - На топ-1 кейс (showcase)
   - На форму заявки (CTA секции)

2. **Из блога:**
   - На релевантные услуги (контекстные ссылки)
   - На related posts (в конце статьи)
   - На лид-магниты (внутри статьи)
   - На форму подписки на email

3. **Со страниц услуг:**
   - На релевантные кейсы (секция на странице)
   - На форму заявки (Primary CTA)
   - На другие услуги (cross-sell)
   - На FAQ
   - На страницу о компании (если упоминается процесс/команда)

4. **Из кейсов:**
   - На релевантные услуги ("Хотите такой же результат?")
   - На форму заявки
   - На другие кейсы (related cases)

5. **Везде:**
   - На контакты (в header, footer, форма на каждой странице)
   - На главную (logo)

## 9. URL СТРУКТУРА

### Принципы:

1. **ЧПУ (человеко-понятные URL):**
   - Только латиница и дефисы
   - Нет спецсимволов, пробелов, подчеркиваний
   - Короткие и описательные

2. **Структура отражает иерархию:**
   - /category/subcategory/page
   - Максимум 3-4 уровня вложенности

3. **SEO-оптимизация:**
   - Включают ключевые слова
   - Без стоп-слов (а, и, или, для, на)
   - Lowercase (маленькие буквы)

4. **Consistency (единообразие):**
   - Один формат для всех URL одного типа

### Примеры URL:

**Главная:**
```
https://example.com/
```

**О компании:**
```
https://example.com/about
https://example.com/about/team
https://example.com/about/history
```

**Услуги:**
```
https://example.com/services
https://example.com/services/service-name-1
https://example.com/services/service-name-2
https://example.com/services/category/service-name
```

**Блог:**
```
https://example.com/blog
https://example.com/blog/category-name
https://example.com/blog/category-name/post-slug
```

**Кейсы:**
```
https://example.com/cases
https://example.com/cases/client-name
https://example.com/cases/industry/client-name
```

**Для e-commerce:**
```
https://example.com/catalog
https://example.com/catalog/category
https://example.com/catalog/category/subcategory
https://example.com/catalog/category/product-name
```
