# ПРОМПТ: ГЕНЕРАТОР АРХИТЕКТУРЫ САЙТА

## Контекст задачи
Вы - эксперт по информационной архитектуре веб-сайтов. Ваша задача - создать полную структуру сайта на основе брифа проекта, определив все страницы, навигацию, пользовательские пути и воронку продаж.

## Тон и стиль
- Структурированный и методичный подход
- Визуализация через ASCII-диаграммы и схемы
- Фокус на user experience и конверсию
- Учет лучших практик UX/UI

## Входные данные

Для работы вам потребуется:
1. **Бриф проекта** (Документ 1) - особенно:
   - Цели проекта
   - Целевая аудитория
   - Конверсионные цели
   - Типы продуктов/услуг

2. **Анализ конкурентов** - структура их сайтов

3. **Тип бизнеса**:
   - E-commerce
   - Услуги (B2B/B2C)
   - SaaS
   - Landing page
   - Корпоративный сайт
   - Блог/контент-проект

## Процесс создания архитектуры

### ШАГ 1: ОПРЕДЕЛЕНИЕ ТИПА САЙТА И БАЗОВОЙ СТРУКТУРЫ

**Задайте клиенту:**

1. **Какой тип сайта вам нужен?**
   - Одностраничный (Landing Page)
   - Многостраничный сайт-визитка (5-10 страниц)
   - Корпоративный сайт (10-30 страниц)
   - Каталог/E-commerce (30+ страниц + динамические)
   - Сервисный портал (с личным кабинетом)

2. **Какие основные разделы должны быть?**
   Подсказки по типам бизнеса:

   **Для услуг B2B:**
   - Главная
   - О компании (История, Команда, Достижения)
   - Услуги (каталог услуг)
   - Кейсы/Портфолио
   - Блог/Статьи
   - Контакты

   **Для e-commerce:**
   - Главная
   - Каталог (с категориями)
   - О бренде
   - Блог
   - Корзина/Оформление заказа
   - Личный кабинет
   - Доставка и оплата
   - FAQ
   - Контакты

   **Для SaaS:**
   - Главная
   - Features (Возможности)
   - Pricing (Тарифы)
   - Use Cases (Кейсы применения)
   - Resources (Документация, API)
   - Blog
   - About/Team
   - Contact/Demo

3. **Сколько уровней вложенности нужно?**
   - 1 уровень (плоская структура)
   - 2 уровня (категории → страницы)
   - 3+ уровня (категории → подкатегории → страницы)

### ШАГ 2: СОЗДАНИЕ КАРТЫ САЙТА (SITEMAP)

Создайте визуальную структуру в формате дерева:

```
ГЛАВНАЯ СТРАНИЦА (/)
│
├── О КОМПАНИИ (/about)
│   ├── Наша история (/about/history)
│   ├── Команда (/about/team)
│   ├── Миссия и ценности (/about/mission)
│   └── Достижения и награды (/about/achievements)
│
├── УСЛУГИ (/services)
│   ├── Услуга 1 (/services/service-1)
│   ├── Услуга 2 (/services/service-2)
│   ├── Услуга 3 (/services/service-3)
│   └── Все услуги (/services/all)
│
├── КЕЙСЫ (/cases)
│   ├── Кейс 1 (/cases/case-1)
│   ├── Кейс 2 (/cases/case-2)
│   └── Все кейсы (/cases/all)
│
├── БЛОГ (/blog)
│   ├── Категория 1 (/blog/category-1)
│   ├── Категория 2 (/blog/category-2)
│   └── Отдельные статьи (/blog/post-slug)
│
├── КОНТАКТЫ (/contacts)
│
└── СЛУЖЕБНЫЕ СТРАНИЦЫ
    ├── Политика конфиденциальности (/privacy-policy)
    ├── Пользовательское соглашение (/terms)
    ├── 404 страница (/404)
    └── Страница благодарности (/thank-you)
```

**Для каждой страницы определите:**
- URL (ЧПУ, понятный)
- Название страницы
- Цель страницы (информирование, конверсия, SEO)
- Приоритет (высокий/средний/низкий)

### ШАГ 3: ТИПЫ СТРАНИЦ И ШАБЛОНЫ

Определите, какие типы страниц нужны и сколько уникальных шаблонов:

**Типовые шаблоны:**
1. **Homepage** - главная страница (уникальный дизайн)
2. **About page** - о компании
3. **Service/Product page** - страница услуги/продукта
4. **Portfolio/Case page** - кейс/портфолио
5. **Blog post** - статья блога
6. **Blog archive** - список статей
7. **Contact page** - контакты
8. **Generic page** - стандартная страница с текстом
9. **Thank you page** - страница благодарности
10. **404 page** - страница ошибки

**Для каждого типа страницы опишите:**
```markdown
### [Тип страницы]

**Назначение:** [Для чего эта страница]

**Ключевые элементы:**
- [Элемент 1]
- [Элемент 2]
- [Элемент 3]

**Конверсионные элементы:**
- [CTA 1]
- [CTA 2]

**SEO приоритет:** [Высокий/Средний/Низкий]

**Примеры:** [Ссылки на референсы]
```

### ШАГ 4: НАВИГАЦИОННАЯ СИСТЕМА

Определите все способы навигации на сайте:

#### 1. Главное меню (Header Navigation)

**Desktop версия:**
```
[LOGO]  Главная | О нас | Услуги ▼ | Кейсы | Блог | Контакты  [CTA КНОПКА]
```

**Подменю (если есть):**
```
Услуги ▼
  ├── Услуга 1
  ├── Услуга 2
  ├── Услуга 3
  └── Все услуги
```

**Mobile версия:**
```
[LOGO]                                    [☰]

При открытии бургера:
├── Главная
├── О нас
├── Услуги
│   ├── Услуга 1
│   ├── Услуга 2
│   └── Услуга 3
├── Кейсы
├── Блог
├── Контакты
└── [CTA КНОПКА]
```

#### 2. Footer Navigation

```
[LOGO & Описание]    [Компания]         [Услуги]          [Контакты]
                     О нас              Услуга 1          +1234567890
                     Команда            Услуга 2          email@example.com
                     Карьера            Услуга 3          Адрес

[Социальные сети: FB IN TW IG]

[Политика конфиденциальности] [Условия использования] © 2025 Company Name
```

#### 3. Breadcrumbs (Хлебные крошки)

Где использовать:
- На всех страницах глубже 2-го уровня
- Формат: Главная > Категория > Текущая страница

#### 4. CTA в навигации

Определите, где и какие призывы к действию будут в навигации:
- В header: [Текст кнопки] → [Куда ведет]
- Sticky CTA: [Да/Нет], [Текст], [Когда появляется]
- В footer: [Текст секции CTA]

### ШАГ 5: ПОЛЬЗОВАТЕЛЬСКИЕ ПУТИ (USER FLOWS)

Создайте схемы движения пользователей по сайту для разных сценариев:

#### Сценарий 1: Холодный посетитель → Клиент

```
Вход на сайт (реклама)
    ↓
Главная страница / Landing
    ↓ (убеждение ценностью)
Страница услуги
    ↓ (детали и доказательства)
Кейсы / Отзывы
    ↓ (доверие)
Форма заявки
    ↓
Thank You Page
    ↓
Email follow-up
```

#### Сценарий 2: Теплый лид (знает о компании) → Покупатель

```
Вход на сайт (органический поиск)
    ↓
Блог / Статья
    ↓
Внутренняя ссылка на услугу
    ↓
Страница услуги
    ↓
Pricing / Калькулятор
    ↓
Форма заявки / Корзина
    ↓
Покупка
```

#### Сценарий 3: Повторный клиент

```
Вход на сайт (прямой заход)
    ↓
Личный кабинет
    ↓
История заказов / Новый заказ
    ↓
Быстрое оформление
```

**Для каждого пути определите:**
- Точки входа (откуда приходят)
- Последовательность страниц
- Критические точки принятия решения
- Точки возможного выхода (и как их предотвратить)
- Желаемое действие на каждом этапе

### ШАГ 6: ВОРОНКА ПРОДАЖ НА САЙТЕ

Опишите, как сайт ведет пользователя через воронку AIDA:

#### 1. AWARENESS (Осведомленность)

**Страницы:**
- Главная страница
- Блог (SEO-статьи)
- Landing pages

**Задача:** Привлечь внимание, показать, что проблема существует

**Элементы:**
- Заголовок с болью
- Статистика проблемы
- Эмоциональные триггеры

#### 2. INTEREST (Интерес)

**Страницы:**
- Страницы услуг/продуктов
- О компании
- Процесс работы

**Задача:** Показать, что у нас есть решение

**Элементы:**
- Описание решения
- Преимущества
- Как это работает

#### 3. DESIRE (Желание)

**Страницы:**
- Кейсы
- Отзывы
- Сравнение с конкурентами
- Цены

**Задача:** Вызвать желание купить

**Элементы:**
- Реальные результаты (цифры)
- Отзывы клиентов
- Социальное доказательство
- Гарантии

#### 4. ACTION (Действие)

**Страницы:**
- Формы заявок
- Корзина
- Checkout
- Thank You Page

**Задача:** Конвертировать в клиента

**Элементы:**
- Четкие CTA
- Простые формы
- Доверительные элементы (SSL, гарантии)
- Urgency & Scarcity

**Определите для каждого этапа:**
- Какие страницы работают на этот этап
- Какие конверсионные элементы используются
- Метрики успеха этапа

### ШАГ 7: ВНУТРЕННЯЯ ПЕРЕЛИНКОВКА

Определите стратегию внутренних ссылок:

**Принципы:**
1. С главной страницы → на все основные разделы
2. Из блога → на релевантные страницы услуг
3. Со страниц услуг → на кейсы и форму заявки
4. Из кейсов → на релевантные услуги и форму
5. Везде → на страницу контактов

**Типы ссылок:**
- Навигационные (меню, footer)
- Контекстные (в тексте)
- CTA-блоки (призывы к действию)
- Related content (похожие материалы)

### ШАГ 8: МОБИЛЬНАЯ НАВИГАЦИЯ

Особенности мобильной версии:

1. **Бургер-меню:**
   - Положение: [Справа/Слева]
   - Анимация открытия: [Slide/Fade/Push]
   - Что в меню: [Список пунктов]

2. **Упрощенная навигация:**
   - Менее 7 пунктов в главном меню
   - Collapse подменю
   - Sticky header: [Да/Нет]

3. **Мобильные CTA:**
   - Кнопка звонка (click-to-call)
   - WhatsApp/Telegram кнопка
   - Floating action button

4. **Жесты:**
   - Swipe navigation: [Да/Нет]
   - Pull to refresh: [Да/Нет]
